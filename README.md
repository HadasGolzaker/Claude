# Voiceprint · טביעת קול

אפליקציה שבונה פרופיל אישיות וכתיבה ומייצרת פוסטים ללינקדאין דרך Claude.
המדריך הזה מסביר, צעד אחר צעד, איך להעלות אותה לאוויר. **לא צריך ידע בתכנות.**

מה שינינו לעומת הקובץ המקורי (רק תשתית — השאלון, הפרומפטים והעיצוב לא נגענו בהם):

1. **מפתח ה-API עבר לשרת.** קודם המפתח היה גלוי בדפדפן; עכשיו הוא שמור בצד שרת ואף אחד לא יכול לגנוב אותו.
2. **התחברות ושמירה אישית.** כל משתמשת נכנסת עם המייל שלה (קישור כניסה, בלי סיסמה), והפרופיל + היסטוריית הפוסטים נשמרים בענן וזמינים מכל מכשיר.
3. **מוכן לפריסה ב-Vercel.**

---

## מה צריך להכין (חשבונות חינמיים)

- חשבון [GitHub](https://github.com) — כדי לאחסן את הקוד.
- חשבון [Vercel](https://vercel.com) — כדי להעלות את האתר לאוויר.
- חשבון [Supabase](https://supabase.com) — למסד הנתונים וההתחברות.
- מפתח Claude API מ-[console.anthropic.com](https://console.anthropic.com).

---

## שלב 1 — Supabase (מסד הנתונים + התחברות)

1. היכנסי ל-[supabase.com](https://supabase.com) ולחצי **New project**. תני שם, בחרי סיסמה למסד (שמרי אותה בצד) ולחצי **Create**. המתיני דקה עד שהפרויקט מוכן.
2. בתפריט השמאלי: **SQL Editor → New query**. הדביקי את כל התוכן של הקובץ `supabase-setup.sql` (נמצא בפרויקט), ולחצי **Run**. זה יוצר את הטבלה ששומרת את הנתונים.
3. בתפריט: **Project Settings → API**. משם תעתיקי שני ערכים (נשתמש בהם בשלב 3):
   - **Project URL** (משהו כמו `https://xxxx.supabase.co`)
   - **anon public** key (מחרוזת ארוכה שמתחילה ב-`eyJ...`)

## שלב 2 — העלאת הקוד ל-GitHub

הכי פשוט: ב-GitHub לחצי **New repository**, תני שם (למשל `voiceprint`), ואז **uploading an existing file** — וגררי את כל הקבצים של הפרויקט הזה. לחצי **Commit**.

> אם אני (Claude) עוזרת לך מתוך GitHub — אני יכולה לדחוף את הקוד לענף שלך במקומך, פשוט בקשי.

---

## שלב 3 — פריסה ב-Vercel

1. היכנסי ל-[vercel.com](https://vercel.com), לחצי **Add New → Project**, וחברי את ה-repository מ-GitHub.
2. לפני שלוחצים **Deploy**, פתחי **Environment Variables** והוסיפי שלושה משתנים:

   | Name | Value |
   |------|-------|
   | `ANTHROPIC_API_KEY` | מפתח ה-Claude שלך (`sk-ant-...`) |
   | `SUPABASE_URL` | ה-Project URL מ-Supabase |
   | `SUPABASE_ANON_KEY` | ה-anon public key מ-Supabase |
   | `GEMINI_API_KEY` | מפתח Google Gemini שלך (ליצירת תמונת הפרופיל - ראו שלב 3.5) |

3. לחצי **Deploy**. אחרי כדקה תקבלי כתובת אתר (משהו כמו `https://voiceprint.vercel.app`).

---

## שלב 3.5 — מפתח Google Gemini (ליצירת תמונת הפרופיל)

כשלא מעלים תמונה בשאלון, האפליקציה מייצרת דמות ויזואלית באמצעות Gemini 2.5 Flash Image (המכונה גם "Nano Banana"). זו עלות זעירה לכל תמונה (כמה סנטים), ודורשת מפתח נפרד מ-Claude:

1. היכנסי ל-[aistudio.google.com/apikey](https://aistudio.google.com/apikey) והתחברי עם חשבון Google.
2. לחצי **Create API key**, ואם מתבקש - צרי/בחרי פרויקט Google Cloud (חינמי).
3. העתיקי את המפתח שנוצר (מתחיל ב-`AIza...`).
4. ב-Vercel: **Settings → Environment Variables**, הוסיפי משתנה בשם `GEMINI_API_KEY` עם הערך שהעתקת.
5. **Deployments → ⋯ → Redeploy** כדי שהשינוי ייכנס לתוקף.

אם המפתח לא מוגדר, תמונת הפרופיל פשוט תישאר עם ראשי התיבות - שאר האפליקציה תמשיך לעבוד רגיל.

---

## שלב 4 — לחבר את ההתחברות לכתובת האתר

כדי שקישור הכניסה במייל יחזיר את המשתמשת לאתר הנכון:

1. ב-Supabase: **Authentication → URL Configuration**.
2. ב-**Site URL** הדביקי את כתובת ה-Vercel שלך (מהשלב הקודם).
3. ב-**Redirect URLs** הוסיפי את אותה כתובת (אפשר גם עם `/*` בסוף).
4. שמרי.

---

## שלב 5 — בדיקה

1. פתחי את כתובת האתר. יופיע מסך כניסה — הכניסי את המייל שלך.
2. בדקי את תיבת המייל ולחצי על קישור הכניסה. תחזרי לאתר מחוברת.
3. עברי את השאלון. הפרופיל שנבנה יישמר אוטומטית, וגם כל פוסט שתייצרי.
4. תתחברי מהטלפון עם אותו מייל — הכל יהיה שם.

זהו. 🎉

## שאלות נפוצות

**האם המפתח שלי בטוח?** כן. הוא שמור רק ב-Vercel (משתני סביבה) ולעולם לא מגיע לדפדפן.

**מישהו יכול לנצל את המפתח?** לא. השרת מאשר בקשות רק למשתמשים מחוברים.

**תמונות גדולות נכשלות ביצירת פוסט.** לשרת של Vercel יש מגבלת גודל בקשה (~4.5MB). אם מעלים כמה תמונות כבדות, כדאי להעלות תמונות קטנות/קלות יותר.

**החלפת מפתח או כתובת Supabase.** משנים את הערך ב-Vercel → Settings → Environment Variables, ואז **Redeploy**. לא נוגעים בקוד.
